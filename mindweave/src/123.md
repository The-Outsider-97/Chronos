# MindWeave × SLAI Agent Integration Plan

## Goal
Select a practical subset of SLAI agents for Project: MindWeave and map them to game systems that are designed for far-transfer outcomes (IQ, executive function, EQ, and metacognition).

## Recommended Agent Stack (what to integrate first)

### 1) CollaborativeAgent (orchestration backbone)
- **Why**: You need one coordinator to route every gameplay event (puzzle actions, dialogue turns, debrief prompts) to the right specialist.
- **How**:
  - Register specialist agents in the Collaboration Manager / task router.
  - Route by `task_type` (`cognitive_puzzle`, `npc_dialogue`, `stress_event`, `debrief_reflection`, `safety_audit`).

### 2) PlanningAgent (dynamic scenario planning)
- **Why**: MindWeave depends on branching quests, changing constraints, and adaptive challenge progression.
- **How**:
  - Use planning memory + probabilistic planner for quest-state transitions.
  - Feed real-time game telemetry (success rate, latency, retries) to planning heuristics.

### 3) ReasoningAgent (novel problem generation + evaluation)
- **Why**: Your IQ engine needs non-trivial, novel logic patterns rather than repeated templates.
- **How**:
  - Use rule/probabilistic reasoning modules to generate and validate procedural logic gate puzzles.
  - Attach solver traces so post-level debrief can explain *why* an answer path worked.

### 4) KnowledgeAgent (world model + narrative consistency)
- **Why**: Faction politics, social memory, and quest dependencies need structured knowledge continuity.
- **How**:
  - Store faction relations, dialogue outcomes, and player behavior signatures in knowledge memory/cache.
  - Use ontology manager to keep narrative entities and constraints coherent.

### 5) PerceptionAgent (multimodal EQ signal ingestion)
- **Why**: EQ mechanics need text/voice/facial-cue fusion for micro-expression and empathy training.
- **How**:
  - Ingest camera/audio/text signals when available.
  - Fuse into a shared embedding and pass to dialogue scoring + adaptive coaching.
  - Keep this optional (graceful fallback to text-only if hardware/permissions unavailable).

### 6) LanguageAgent (LLM-driven NPC dialogue loop)
- **Why**: Empathy Bridge and negotiation systems require nuanced, open-ended language interaction.
- **How**:
  - Use NLU for intent/emotion parsing and NLG for contextual NPC responses.
  - Add rubric-based scoring (validation, perspective-taking, de-escalation, repair attempts).

### 7) ExecutionAgent (tool/API action layer)
- **Why**: You need a safe bridge between high-level plans and concrete operations (state writes, API calls, triggers).
- **How**:
  - Gate all external actions (leaderboard write, biometric update, progression unlock) through execution validator + recovery.

### 8) SafetyAgent + AlignmentAgent + EvaluationAgent (trust layer)
- **Why**: This is essential for emotional simulations, player wellbeing, and LLM behavior governance.
- **How**:
  - Safety checks on every high-impact interaction.
  - Alignment monitor for fairness/bias/ethical constraints/counterfactual auditing.
  - Evaluation agent tracks quality, drift, and regressions over time.

### 9) LearningAgent + Adaptive components (difficulty calibration)
- **Why**: Your far-transfer target depends on sustained challenge in the zone of proximal development.
- **How**:
  - Start with RL/DQN style adaptation for cognitive challenge tuning.
  - Add RSI/meta-learning later for personalized long-horizon adaptation.

## Priority order (phased delivery)

### Phase 1 (MVP)
- CollaborativeAgent
- PlanningAgent
- ReasoningAgent
- LanguageAgent
- SafetyAgent
- EvaluationAgent

### Phase 2
- KnowledgeAgent
- ExecutionAgent
- Adaptive/Learning modules for dynamic overload

### Phase 3
- PerceptionAgent multimodal stack
- AlignmentAgent deep auditing (full fairness and counterfactual checks)
- Advanced meta-learning (MAML/RSI) for personalization

## Mapping SLAI agents to MindWeave mechanics

- **Temporal Hacking (Dual N-Back)**: Learning/Adaptive + Planning + Evaluation.
- **Supply Chain Puzzles**: Planning + Reasoning + Knowledge.
- **Procedural Logic Gates**: Reasoning + Learning.
- **Micro-Expression Interrogations**: Perception + Language + Safety.
- **Empathy Bridge**: Language + Knowledge + Alignment.
- **Biometric Regulation**: Perception + Execution + Safety.
- **Metacognitive Debriefing**: Language + Knowledge + Evaluation.

## Practical integration contract (event schema)

Define a common event envelope all agents can consume:
- `session_id`
- `player_id`
- `task_type`
- `game_state_snapshot`
- `telemetry` (reaction time, retries, confidence, HRV if present)
- `safety_context`
- `requested_action`

This lets CollaborativeAgent route consistently and allows Safety/Evaluation to run as universal middleware.

## Key implementation cautions
- Keep biometric and affective inference opt-in and privacy-first.
- Use age-aware and vulnerability-aware safety policies for socio-emotional content.
- Avoid over-optimizing one metric (e.g., speed) at the expense of reflection quality and emotional regulation.
- Log adaptation rationale to support player trust and scientific interpretability.

## Bottom line
For MindWeave, integrate SLAI as a **coordinated multi-agent system**, not as isolated features. Start with orchestration, planning/reasoning, language, and safety; then add knowledge, adaptation, and multimodal perception for deeper far-transfer potential.
